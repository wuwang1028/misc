# 003-Web管理员账户入侵日志分析复现方案

本题属于离线日志关联取证题，复现重点是还原“Webshell 交互导致数据库管理员密码被篡改”的完整证据链，而不是搭建完整业务系统。

## 一、复现目标

复现时应证明：在同一时间窗口内，外部来源 `157.7.196.221` 多次请求可疑脚本 `/uploads/upload/xiaoma.php`，随后数据库日志出现 `update admin set password="154444" where id=1`，因此该来源可被认定为导致后台管理员账户异常的攻击者。

## 二、最小复现材料

| 材料 | 用途 |
| --- | --- |
| Web 访问日志 | 提取请求方法、来源 IP 与访问路径 |
| 数据库日志 | 定位管理员密码异常更新语句 |
| 时间线对照方法 | 建立攻击入口与异常结果的因果关系 |

## 三、建议复现步骤

首先在数据库日志中搜索 `update admin`、`password` 等敏感关键字，定位后台账户异常修改时刻。随后在同一时间窗口附近筛查 Web 访问日志中的 POST 请求、可疑参数以及非常规 PHP 路径访问。最后把请求时间与数据库异常时间进行比对，保留能够解释异常结果的最直接来源地址。

## 四、预期复现结论

若输入日志与题目附件一致，应稳定得到以下提交结果：

```text
flag{157.7.196.221}
```
