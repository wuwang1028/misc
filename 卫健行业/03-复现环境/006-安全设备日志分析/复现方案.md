# 006-安全设备日志分析复现方案

本题属于离线数据库日志取证题，复现重点是从 SQL Server 日志中恢复攻击路径，而不是重建完整业务系统。

## 一、复现目标

复现时应证明：客户端 `192.168.75.139` 在异常操作中持续出现，并触发了 `show advanced options` 与 `xp_cmdshell` 的配置变更，因此题目所需拼接结果应为攻击 IP 加 `sp_configure 'show advanced options', 1`。

## 二、最小复现材料

| 材料 | 用途 |
| --- | --- |
| 原始 `sqlDB.log` | 记录客户端地址与配置变更事件 |
| 编码转换工具 | 将 UTF-16LE 日志转为 UTF-8 以便检索 |
| 文本检索工具 | 搜索 `客户端`、`show advanced options`、`xp_cmdshell` 等关键字 |

## 三、建议复现步骤

首先将原始 SQL Server 日志转换为 UTF-8 文本；随后检索全部客户端地址，筛出与异常配置修改同一时间窗口内出现最多的可疑来源；再定位 `Configuration option 'show advanced options' changed from 0 to 1` 与 `Configuration option 'xp_cmdshell' changed from 0 to 1` 两条记录；最后把攻击 IP 与关键命令按题目格式拼接，并将空格替换为下划线。

## 四、预期复现结论

若输入日志与题目附件一致，应稳定得到以下答案：

```text
flag{192.168.75.139+sp_configure_'show_advanced_options',_1}
```
