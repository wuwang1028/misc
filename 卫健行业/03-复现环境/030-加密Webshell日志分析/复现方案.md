# 030-加密Webshell日志分析：复现方案

本题属于访问日志取证分析题，复现重点是从成功利用流量中识别真实 Webshell 密码，而不是重新执行攻击。

## 一、复现目标

从 `access.log` 中定位真实攻击使用的单行 Webshell 入口，并恢复其固定口令参数名。

## 二、推荐步骤

| 步骤 | 操作说明 |
| --- | --- |
| 1 | 对访问日志进行结构化解析，统计来源 IP、URI、状态码与请求方法 |
| 2 | 排除大规模扫描阶段的 `404/302` 噪声，聚焦返回 `200` 的可疑 PHP 文件 |
| 3 | 锁定 `/font.php` 与 `/lang.php` 两个高可疑入口 |
| 4 | 对 `/font.php` 的请求体做对比，提取所有请求中恒定不变的前置参数名 |
| 5 | 验证其后承载的 PHP 代码功能不同，但前置参数名始终一致，从而认定其为 Webshell 密码 |
