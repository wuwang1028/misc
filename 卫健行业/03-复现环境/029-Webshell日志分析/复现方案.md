# 029-Webshell日志分析：复现方案

本题属于日志取证分析题，复现重点在于**如何从混杂的系统调用日志中提取两段有效 flag**，而不是重新执行攻击行为。建议在隔离环境中对原始日志进行只读分析。

## 一、复现目标

从 `sys.log` 中恢复 `flag1` 和 `flag2` 的原始编码内容，分别解码后按题面要求拼接成最终提交值。

## 二、推荐步骤

| 步骤 | 操作说明 |
| --- | --- |
| 1 | 对 `sys.log` 做关键词检索，先定位包含 `flag`、`secret`、`token`、Base64 特征串的日志行 |
| 2 | 过滤掉大量 `flags=` 文件打开噪声，保留与 `flag` 语义真正相关的事件 |
| 3 | 提取恶意计划任务中的 Base64 载荷并解码，得到 `flag1{vj53msxo5zb8kbhc}` |
| 4 | 提取数据库查询结果中的 Base64 字符串 `ZmxhZzJ7em44b3BuemtiYWJnYXR1ZX0=` 并解码，得到 `flag2{zn8opnzkbabgatue}` |
| 5 | 按题面要求去掉两段外层嵌套后直接拼接 |

## 三、预期结果

完成上述步骤后，应得到：

```text
flag{vj53msxo5zb8kbhczn8opnzkbabgatue}
```
