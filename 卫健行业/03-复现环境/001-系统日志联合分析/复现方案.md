# 001-系统日志联合分析复现方案

本题属于典型的离线日志取证题，不依赖在线服务环境即可复现。复现时只需准备包含 SSH 认证记录的 `auth.log` 或同类日志文件，并保证日志中同时包含失败认证、认证次数异常以及成功登录事件。

## 一、复现目标

复现的目标不是重新搭建完整业务主机，而是验证以下分析结论是否能够从日志中稳定推出：攻击者 IP `177.77.135.158` 在短时间内多次尝试 `root` 口令，并最终成功登录，因此应被认定为可疑入侵者。

## 二、最小复现材料

| 材料 | 用途 |
| --- | --- |
| SSH 认证日志样本 | 承载失败、限制与成功登录记录 |
| 基础文本检索工具 | 用于按 IP 与关键字筛查 |
| 时间线比对方法 | 用于串联连续事件 |

## 三、建议复现步骤

首先筛查全部 `Accepted password` 事件，枚举所有成功登录来源；随后对每个来源回溯同一时间段的 `Failed password`、`maximum authentication attempts exceeded` 等关键字段；最后保留同时满足“连续失败后成功登录”的地址作为结果。

## 四、预期复现结论

若输入日志与题目附件一致，则应稳定得到以下结论：

```text
flag(177.77.135.158)
```
