# 008-HTTP流量文件提取复现方案

本题属于离线流量取证题，复现重点是从 HTTP 流量中恢复被传输对象的真实内容，并据此确认数据库名称。

## 一、复现目标

复现时应证明：在 `wenjiantiqu.pcapng` 中存在通过 HTTP 返回的 ZIP 压缩对象，该对象内部含有数据库导出文件 `wstmart.sql`，因此被窃取数据库名称为 `wstmart`。

## 二、最小复现材料

| 材料 | 用途 |
| --- | --- |
| `pcapng` 流量文件 | 提取 HTTP 请求、响应与传输对象 |
| HTTP 解析脚本或抓包工具 | 重组会话与提取响应对象 |
| 二进制字符串检索工具 | 从原始对象中定位 `PK` 与 `.sql` 文件名 |

## 三、建议复现步骤

首先对流量文件进行 HTTP 会话解析，确认是否存在文件下载类响应；随后筛查响应头中的 `Content-Type: application/zip` 等特征；再对响应体或原始流量内容做二进制可读检索，定位 ZIP 文件头与内部文件名；最后将恢复出的 `wstmart.sql` 去掉扩展名，得到数据库名 `wstmart`。

## 四、预期复现结论

若输入流量与题目附件一致，应稳定得到以下提交结果：

```text
flag{wstmart}
```
