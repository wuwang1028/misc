# 主机基线加固复现方案

## 一、复现目标

本题的复现目标不是立即交付可运行容器，而是先明确环境构成、判题逻辑与实施步骤，为后续真正制作训练环境提供准确蓝图。该题需要复现的核心能力包括：提供一个可通过 ttyd 登录的 Linux 主机；预置一个“尚未满足题目要求”的 PAM 密码复杂度配置；由裁判脚本定期检测配置内容；当检测结果满足规则后，在系统内生成 `/flag`。

## 二、建议环境架构

建议后续采用单容器或单虚拟机的轻量结构进行复现。由于题目本身逻辑简单，不依赖多机交互，因此没有必要拆分多个服务。推荐结构如下。

| 组件 | 作用 | 建议实现 |
| --- | --- | --- |
| Ubuntu 20.04 基础系统 | 与原题环境保持一致 | Docker 或轻量虚拟机 |
| ttyd | 提供 Web 终端入口 | 前台启动 shell 或 login 程序 |
| ctf 用户 | 选手登录账号 | 用户名密码均设为 `ctf` |
| PAM 配置文件 | 提供待修复的密码复杂度策略 | `/etc/pam.d/common-password` |
| 判题脚本 | 定时检查配置并写出 `/flag` | shell 脚本 + cron 或 while 循环 |

## 三、系统初始状态设计

为了让题目具备可操作性，初始环境应刻意设置为“部分合规但未完全满足要求”的状态。具体来说，`/etc/pam.d/common-password` 中密码策略建议预置为：

```text
password requisite pam_cracklib.so retry=3 minlen=8 difok=3
```

这样既能让参赛者看出系统已经开启了密码复杂度控制，又不会直接满足题面要求，从而保留分析与修复空间。

同时，建议把该文件权限显式设置为对 `ctf` 用户可写，例如通过组权限、ACL 或直接放宽文件权限实现。原题中该文件实际呈现为可写状态，这也是参赛者不借助 sudo 完成修改的关键路径。

## 四、判题脚本逻辑设计

判题逻辑建议做成一个极简轮询脚本，每隔 10 至 15 秒检测一次 `/etc/pam.d/common-password` 中 `pam_cracklib.so` 这一行是否同时包含以下要素：

| 检查项 | 目标值 |
| --- | --- |
| `minlen` | `8` |
| `ucredit` | `-1` |
| `lcredit` | `-1` |
| `dcredit` | `-2` |
| `ocredit` | `-1` |

如果全部命中，则创建 `/flag` 并写入预置 flag 字符串。判题脚本可以采用如下思路：

```bash
#!/bin/bash
while true; do
  line=$(grep 'pam_cracklib.so' /etc/pam.d/common-password 2>/dev/null)
  if echo "$line" | grep -q 'minlen=8' \
    && echo "$line" | grep -q 'ucredit=-1' \
    && echo "$line" | grep -q 'lcredit=-1' \
    && echo "$line" | grep -q 'dcredit=-2' \
    && echo "$line" | grep -q 'ocredit=-1'; then
      echo 'xctf{REPLACE_WITH_REAL_FLAG}' > /flag
      chmod 777 /flag
      break
  fi
  sleep 15
done
```

这一判题方式与原题实际行为高度一致，且便于维护。

## 五、ttyd 与登录环境设计

原题使用 ttyd 暴露终端，并允许通过 `ctf/ctf` 登录系统。复现时建议：

1. 创建 `ctf` 用户并设置初始密码为 `ctf`；
2. 确保容器或虚拟机具备 PAM 登录链；
3. 使用 ttyd 暴露 login 终端，或暴露 bash 并在题目说明中引导参赛者使用系统 shell；
4. 在 `~/README.md` 中写入题目要求与判题说明。

README 可直接沿用下列思路：

> 请设置主机密码复杂度，要求：设置密码长度最少 8 位，至少包含一个大写字母、一个小写字母、两个数字、一个特殊符号。裁判机每隔 15 秒会判断是否成功加固，若成功加固，会生成 `/flag` 文件。

## 六、参赛者预期解法

从训练设计角度看，参赛者的标准解法应包括以下几个步骤：先登录 ttyd；再阅读 `README.md`；随后检查 `/etc/pam.d/common-password`；识别当前仅设置了 `minlen=8` 而未设置字符类别数量；然后补充 `ucredit=-1`、`lcredit=-1`、`dcredit=-2`、`ocredit=-1`；最后等待判题脚本生成 `/flag`。

这类解法路径清晰，既能训练基础 Linux 安全配置能力，又不会依赖额外的漏洞利用知识。
