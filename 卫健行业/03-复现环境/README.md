# 复现环境目录说明

本目录用于按题目统一归档复现环境材料。每一道环境类题目的 Docker 环境、初始化脚本、修复脚本、验证脚本与单题说明，都会放入对应题目目录中，并与 `../02-题目分析与WP/` 下的同名题目目录形成一一对应关系。

| 题目目录 | 当前状态 |
|---|---|
| `027-安全漏洞事件分析/` | 已补齐为可运行的源码级复现目录，包含 Dockerfile、容器编排、应用源码与基础验证脚本 |
| `035-加密Webshell后门流量分析/` | 附件分析题，无需 Docker 环境；已放入复现方案文档与解密脚本说明 |
| `慢SQL优化/` | 已预留目录 |
| `主机基线加固/` | 已预留目录 |
| `系统后门排查/` | 已放入复现方案文档，后续继续补齐自动化环境材料 |
| `服务后门排查/` | 已放入复现方案文档，后续继续补齐 Docker 环境与初始化材料 |
| `Web服务故障处理/` | 已放入复现方案文档，后续继续补齐 Docker 环境、FPM 配置模板与初始化材料 |
| `Web漏洞修复/` | 已预留目录 |
| `数据库安全加固/` | 已预留目录 |
| `系统服务加固/` | 已预留目录 |

调整后的结构原则是：**单题分析统一进入 `02-题目分析与WP/`，单题复现统一进入 `03-复现环境/`**。其中，若某题已经具备较完整源码与验证材料，则应优先落地为可运行复现目录，而不是长期停留在“方案待补齐”状态。
