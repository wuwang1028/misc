# 010-WEB系统已知漏洞攻击流量分析：复现方案

本题属于流量取证分析题，复现重点不在于重新攻击真实系统，而在于**复盘攻击者如何从 HTTP 流量中一步步获取秘密**。建议在隔离环境中使用原始流量包与解析脚本进行只读分析，避免对外发起任何真实攻击请求。

## 一、复现目标

在本地还原 HTTP 请求与响应，找到攻击者成功利用的命令执行接口，并从返回内容中恢复 `secret.log` 的编码值，最终解码得到 flag。

## 二、推荐步骤

| 步骤 | 操作说明 |
| --- | --- |
| 1 | 使用脚本对 `HTTP攻击流量.pcapng` 做 HTTP 会话重组，提取请求与响应对 |
| 2 | 过滤所有 `404 Not Found` 与默认欢迎页，重点关注真正返回有效内容的 `200 OK` 请求 |
| 3 | 锁定 `/atack?cmd=` 相关请求，确认攻击者具备命令执行能力 |
| 4 | 顺着 `ls`、`ls -la ./logs`、`cat secret.log` 这一链条恢复攻击者操作路径 |
| 5 | 对返回的 `JV4SA43FMNZGK5BANFZSAZTMMFTXWVDIGFZV6MLTL5WXSX3TGNRXEZLUL4ZDAMRTL5GDSTC7K4YW47I=` 做 Base32 解码 |
| 6 | 从解码结果中提取最终 flag |

## 三、预期结果

完成上述步骤后，应得到明文：

```text
My secret is flag{Th1s_1s_my_s3cret_2023_L9L_W1n}
```

据此提交：

```text
flag{Th1s_1s_my_s3cret_2023_L9L_W1n}
```
