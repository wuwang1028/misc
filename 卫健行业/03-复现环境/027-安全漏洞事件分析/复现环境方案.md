# 027-安全漏洞事件分析复现环境方案

## 一、调整目标

`027-安全漏洞事件分析` 原先在仓库中只有一份偏方案化的说明文档，和后续题目相比，缺少已经落地的复现环境骨架；与此同时，`02-题目分析与WP/027-安全漏洞事件分析/` 下又已经保存了较完整的源码恢复结果、利用脚本与验证证据。因此，这一题更合理的状态不应继续停留在“后续补齐”，而应直接整理成**与当前仓库统一结构相匹配的已落地复现目录**。

本次调整后的目标有两个。第一，把 `02-题目分析与WP/027-安全漏洞事件分析/` 重组为和后续题目一致的 `01-分析与WP/`、`02-证据与脚本/` 双层结构。第二，把 `03-复现环境/027-安全漏洞事件分析/` 从单一文档目录补齐为**可运行源码 + 容器化启动材料 + 说明文档**的标准复现环境目录。

## 二、题目核心机制

从已恢复的真实源码可以确认，本题并不是单点漏洞，而是由多个设计缺陷拼接出来的一条利用链。

| 环节 | 保留内容 | 作用 |
| --- | --- | --- |
| 文件上传 | `upload.php` 仅按后缀校验图片 | 允许上传 PHAR/JPG 多态文件 |
| 触发点 | `file.php` 中的 `file_exists($_GET['c'])` | 处理 `phar://` 路径时触发 PHAR 元数据反序列化 |
| POP 链 | `xxxx.php` 中的 `Show` / `Content` / `Action` | 反序列化后可落到 `include($this->path)` |
| 会话机制 | `lib/session.php` 的客户端签名 Cookie | 允许围绕 `theme` 字段组织后续包含链 |
| 最终包含点 | `index.php` 中的 `include($session->get('theme', 'light') . '.css')` | 将路径控制能力转化为执行能力 |

因此，本地复现环境只要保留这些关键部件，就能够较稳定地呈现题目的训练价值。是否采用与远端完全相同的 Web 容器并不是第一优先级，真正重要的是把**上传、反序列化、包含、会话伪造、读旗**这条主链闭环保留下来。

## 三、当前落地结构

调整后，本题复现环境目录采用如下结构。

| 路径 | 内容 |
| --- | --- |
| `Dockerfile` | 复现镜像构建文件 |
| `docker-compose.yml` | 本地启动入口 |
| `docker/entrypoint.sh` | 初始化上传目录、写入 `/flag.txt` |
| `docker/php.ini` | PHP 上传与运行时配置 |
| `app/` | 按已恢复源码整理出的完整站点 |
| `app/lib/` | 会话与工具函数 |
| `app/uploads/` | 上传目录 |
| `README.md` | 启动方式、目录说明与使用指引 |
| `init/` | 预留初始化材料 |
| `solution/` | 预留修复与验证材料 |

这套组织方式和仓库中其他题目的“单题目录承载完整复现材料”的方向保持一致，也把 027 从“只有说明文档”提升为“已经有基础落地形态”的状态。

## 四、源码还原原则

本次落地没有凭空重写业务，而是优先使用 `02-题目分析与WP/027-安全漏洞事件分析/02-证据与脚本/` 中已经恢复出的源码。对应关系如下。

## 五、运行与验证

在目录 `03-复现环境/027-安全漏洞事件分析/` 下执行：

```bash
docker compose up --build -d
```

启动后，访问 `http://127.0.0.1:8080/` 即可进入靶场首页。容器启动时会自动写入 `/flag.txt`，其值与 WP 中已验证成功的 flag 保持一致，方便后续验证攻击链是否已经完整打通。

建议至少完成以下观察项：

| 验证项 | 预期现象 |
| --- | --- |
| 首页可访问 | 能看到头像上传器页面 |
| `theme.php` | 可在浅色与深色主题间切换 |
| `upload.php` | 能上传图片后缀文件，并在提示中看到保存路径 |
| `file.php?c=/etc/passwd` | 会返回文件存在性判断结果 |
| 上传 PHAR/JPG 后访问 `phar://...` | 能保留题目原始反序列化触发条件 |

## 六、结构调整结论

这次调整的重点不是“把 027 再写一篇 WP”，而是把它从早期零散归档状态，整理成与后续题目一致的统一结构：分析材料放到 `01-分析与WP/` 与 `02-证据与脚本/` 中，复现材料放到 `03-复现环境/027-安全漏洞事件分析/` 下的可运行目录中。这样后续无论是补充 Docker 细节、增加自动验证脚本，还是继续扩展更完整的利用脚本，都有清晰且一致的落点。
