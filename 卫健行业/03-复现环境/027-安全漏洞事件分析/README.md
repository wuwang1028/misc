# 027-安全漏洞事件分析复现环境

本目录不再只是“方案占位”，而是已经根据 `02-题目分析与WP/027-安全漏洞事件分析/` 中恢复出的真实源码，整理为一个可以直接本地启动的最小复现靶场。其目标不是一比一复制远端基础设施，而是尽量保留题目的**核心攻击闭环**：图片上传、`phar://` 触发、PHAR 元数据反序列化、POP 链到本地文件包含、客户端签名 Cookie 会话，以及基于 `theme` 的最终包含点。

## 目录结构

| 路径 | 作用 |
| --- | --- |
| `app/` | 根据已恢复源码整理出的可运行 PHP 站点 |
| `docker/` | 启动脚本与 PHP 配置 |
| `docker-compose.yml` | 本地一键启动入口 |
| `init/` | 预留初始化材料目录 |
| `solution/` | 预留修复与验证脚本目录 |
| `复现环境方案.md` | 题目设计、利用链与结构说明 |

## 启动方式

在当前目录执行：

```bash
docker compose up --build -d
```

默认会将靶场暴露到本机 `8080` 端口，对应访问地址为：

```text
http://127.0.0.1:8080/
```

## 关键还原点

| 组件 | 说明 |
| --- | --- |
| `app/index.php` | 保留首页注释、主题包含点与客户端会话逻辑 |
| `app/file.php` | 保留 `file_exists($_GET['c'])` 触发点 |
| `app/xxxx.php` | 保留题目 POP 链与 `include($this->path)` sink |
| `app/upload.php` | 保留仅按后缀校验的上传逻辑 |
| `app/lib/session.php` | 保留 bcrypt 签名型客户端 Cookie 会话 |
| `/flag.txt` | 容器启动时写入固定 flag，便于验证最终利用成功 |

## 使用说明

本环境主要用于理解与验证题目结构，不直接附带自动攻击脚本。若需要完整利用，可参考 `../../02-题目分析与WP/027-安全漏洞事件分析/02-证据与脚本/` 中的源码恢复记录、PHAR 构造脚本和相关验证输出。
