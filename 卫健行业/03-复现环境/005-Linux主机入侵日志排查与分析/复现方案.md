# 005-Linux主机入侵日志排查与分析复现方案

本题属于离线主机与 Web 联合日志取证题，复现重点是还原攻击源、确认成功登录端口，并计算题目要求的 `MD5(ip:port)`。

## 一、复现目标

复现时应证明：攻击者来源为 `192.168.203.1`，其最早成功建立的 root SSH 会话端口为 `42998`，因此最终提交值为 `flag{0b97b3c8fbc9941a7aef456b93b4786d}`。

## 二、最小复现材料

| 材料 | 用途 |
| --- | --- |
| `secure` | 提取成功 SSH 登录记录与来源端口 |
| `access.log` | 验证同源 Web 探测与漏洞利用行为 |
| `spring.log` | 确认 SQL 注入写 Webshell 等高危操作 |
| `.bash_history` | 辅助判断攻击者已进行后续落地操作 |

## 三、建议复现步骤

首先在 `access.log` 与 `spring.log` 中锁定持续发起漏洞利用和写 Webshell 的来源地址 `192.168.203.1`；随后在 `secure` 中筛出该地址的全部 `Accepted password for root` 记录；按时间排序后选取最早一次成功登录，即 `Oct 30 12:04:01` 的 `port 42998`；最后将字符串 `192.168.203.1:42998` 做 MD5 计算，得到题目要求的结果。

## 四、预期复现结论

若输入日志与题目附件一致，应稳定得到以下答案：

```text
flag{0b97b3c8fbc9941a7aef456b93b4786d}
```
