# 服务后门排查证据说明

本目录用于保存《服务后门排查》题目的关键取证输出、辅助执行脚本与截图识别结果。为了便于后续复盘，现将各文件的作用整理如下。

| 文件名 | 作用 | 说明 |
| --- | --- | --- |
| `cron_newscan.out` | 首轮环境扫描输出 | 包含当前用户、`crontab -l`、`/etc/crontabs/`、`.ash_history`、`/readflag` 与 `/flag` 初始状态 |
| `cron_compare.out` | 计划任务对比输出 | 用于确认当前用户 crontab 与 `/etc/crontabs/ctf` 中的恶意任务内容一致 |
| `cron_fix_wait_new.out` | 最终修复与等待输出 | 记录清理恶意计划任务、处理历史文件并等待判题后的结果 |
| `cron_postcheck.out` | 修复后状态复查输出 | 明确显示 `.ash_history -> /dev/null`、`crontab -l` 为空、`/flag` 已生成以及最终 flag 内容 |
| `second_challenge_image_findings.md` | 题目截图识别结果 | 记录题名、题面文字、网络配置、主机节点与原始实例入口示例 |
| `service_backdoor_exec.py` | ttyd WebSocket 辅助执行脚本 | 用于在题目在线终端中自动登录并执行命令，保留了解题过程的操作方式 |

从这些证据可以还原完整解题链路：题面明确指出异常来源是定时任务；首轮扫描定位到 Bash 反连型恶意 crontab；后续通过清空用户计划任务并处理 `.ash_history` 实现环境修复；最终由裁判异步生成 `/flag`，从而完成取旗。
