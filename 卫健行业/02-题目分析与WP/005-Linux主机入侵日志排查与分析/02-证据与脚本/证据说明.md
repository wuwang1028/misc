# 005-Linux主机入侵日志排查与分析证据说明

本题取证重点是从多类日志中先锁定攻击者来源 IP，再从认证日志中选取题目要求的 `ip:port` 组合并计算 MD5。

## 关键日志证据

```text
Oct 30 12:04:01 localhost sshd[3312]: Accepted password for root from 192.168.203.1 port 42998 ssh2
Oct 30 12:04:27 localhost sshd[3312]: pam_unix(sshd:session): session opened for user root by (uid=0)

spring.log 中存在：union select ... into outfile '/var/www/html/404.php'
access.log 中同源地址 192.168.203.1 发起大量漏洞探测与利用请求
```

## 结论摘要

| 要素 | 结论 |
| --- | --- |
| 攻击来源 IP | `192.168.203.1` |
| 首次成功 SSH 登录端口 | `42998` |
| 参与哈希计算的字符串 | `192.168.203.1:42998` |
| MD5 | `0b97b3c8fbc9941a7aef456b93b4786d` |
| 最终答案 | `flag{0b97b3c8fbc9941a7aef456b93b4786d}` |

## 说明

虽然攻击者后续还有多个成功 SSH 会话端口，但从题意“攻击者恶意 IP”与单值答案逻辑出发，选择日志中**最早成功建立的入侵会话**最为稳妥，因此采用 `42998` 作为最终端口。
