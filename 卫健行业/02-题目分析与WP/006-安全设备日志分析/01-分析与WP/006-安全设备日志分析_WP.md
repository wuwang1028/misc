# 006-安全设备日志分析 WP

## 一、题目信息与最终结论

本题要求依据给出的 SQL Server 日志文件分析攻击者的攻击路径，并提交 **`flag{攻击IP+显示高级设置的命令}`**，其中空格需要替换为下划线。对原始 `sqlDB.log` 完成编码转换并提取关键配置变更记录后，可以确认最核心的攻击来源地址为 **`192.168.75.139`**，而题目要求的“显示高级设置的命令”应为 **`sp_configure 'show advanced options', 1`**。因此当前最终提交为：

```text
flag{192.168.75.139+sp_configure_'show_advanced_options',_1}
```

本题的分析重点在于把两类证据结合起来：一类是来源于 SQL Server 日志中的客户端地址，另一类是配置项被打开的执行轨迹。只有将二者统一起来，才能同时回答“谁在攻击”和“攻击者执行了什么关键配置命令”这两个问题。

## 二、分析方法

原始日志为 UTF-16LE 编码，如果直接检索，很多关键字符串会因为编码问题难以稳定提取。因此我先把 `sqlDB.log` 转换为 UTF-8 文本，再围绕 `show advanced options`、`xp_cmdshell`、`客户端` 等关键字做检索与上下文还原。

完成转换后，可以清楚看到日志不仅记录了大量来自同一地址的失败连接或异常操作，还记录了 SQL Server 配置项被依次开启的过程。这说明攻击者并非只是在做无效探测，而是在逐步打开 SQL Server 的高危能力，最终形成可执行系统命令的攻击路径。

## 三、攻击来源识别

阶段性结论表明，日志中绝大多数失败登录与危险操作上下文都标记为 **`[客户端: 192.168.75.139]`**。这使得 `192.168.75.139` 成为最稳定、最直接的攻击来源地址。

| 取证要素 | 观察结果 | 结论 |
| --- | --- | --- |
| 客户端地址 | 多数异常记录标记为 `[客户端: 192.168.75.139]` | 攻击行为集中来自该地址 |
| 危险配置变更窗口 | 与同一时间段的高危命令操作相邻 | 不是普通访问，而是攻击链的一部分 |
| 题目所需攻击 IP | 需与关键命令拼接提交 | 选定为 `192.168.75.139` |

因此，本题的攻击者 IP 可以确定为 `192.168.75.139`。

## 四、关键命令还原

日志中最重要的两条配置变更记录如下。

```text
Configuration option 'show advanced options' changed from 0 to 1
Configuration option 'xp_cmdshell' changed from 0 to 1
```

这两条记录揭示了非常典型的 SQL Server 攻击路径。攻击者先打开高级选项，再启用 `xp_cmdshell`，以便后续通过数据库执行系统命令。按照 SQL Server 的常规配置方式，触发第一条配置变更的核心命令应为：

```sql
sp_configure 'show advanced options', 1
```

如果带上常见执行前缀，也可能写作 `exec sp_configure 'show advanced options', 1`。但从题面措辞“显示高级设置的命令”来看，通常取核心命令本体即可，而不必额外加入 `exec`。同时，题目要求空格改为下划线，所以格式化后应写作：

```text
sp_configure_'show_advanced_options',_1
```

## 五、攻击路径说明

从日志呈现的顺序来看，攻击者的动作并不是随意试探，而是有明确目标的数据库利用流程。首先，攻击来源 `192.168.75.139` 在日志中持续出现；随后，`show advanced options` 被打开；接着，`xp_cmdshell` 也被启用。这说明攻击者已经具备执行数据库配置命令的权限，并试图把 SQL Server 从“数据库服务”扩展成“命令执行入口”。

| 阶段 | 关键动作 | 安全意义 |
| --- | --- | --- |
| 第一步 | 识别到异常客户端 `192.168.75.139` | 锁定攻击来源 |
| 第二步 | 执行 `sp_configure 'show advanced options', 1` | 打开高级配置能力 |
| 第三步 | 启用 `xp_cmdshell` | 为系统命令执行做准备 |

这条路径与真实攻防中常见的 MSSQL 入侵手法高度一致，因此结论具有较强稳定性。

## 六、最终答案

综合攻击来源与关键命令还原结果，本题最终提交为：

```text
flag{192.168.75.139+sp_configure_'show_advanced_options',_1}
```
