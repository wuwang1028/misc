# 003-Web管理员账户入侵日志分析 WP

## 一、题目信息与最终结论

本题要求结合站点 Web 日志与数据库日志，找出导致管理员账户异常的攻击者 IP。根据题面截图恢复结果，提交格式为 **`flag{IPv4地址}`**。对附件中的 `access.log` 与数据库日志 `log-T-2601.log` 联合分析后，可以确认导致后台管理员密码被异常篡改的来源地址为 **`157.7.196.221`**，最终提交内容为：

```text
flag{157.7.196.221}
```

本题的关键并不只是发现数据库里出现了异常 `update admin set password=...` 语句，更重要的是将这一异常更新时间与 Web 侧的攻击请求对应起来。经过时间线比对，来源 `157.7.196.221` 对 `/uploads/upload/xiaoma.php` 的多次 POST 请求与数据库异常写操作高度重合，形成了完整而直接的攻击证据链。

## 二、分析思路

这类题目通常要同时回答两个问题。第一，数据库层面到底发生了什么异常；第二，这个异常是由哪个外部请求触发的。如果只看 Web 日志，容易把普通访问、扫描行为和真正的入侵动作混在一起；如果只看数据库日志，又只能看到结果，不能还原攻击入口。因此必须把两类日志统一放进同一时间轴中分析。

我首先在数据库日志中筛查敏感操作关键字，定位到异常管理员密码更新语句；随后回到该时间窗口附近筛查 Web 请求，并剔除静态资源访问与普通页面浏览，只保留动态请求、POST 请求以及可疑脚本访问。最终，`157.7.196.221` 发往 `/uploads/upload/xiaoma.php` 的连续 POST 行为最能解释管理员口令被篡改的成因。

## 三、数据库侧关键证据

数据库日志中最核心的一条异常语句如下。

```text
230927 16:37:10 ... update admin set password="154444" where id=1
```

这条语句清楚表明后台 `admin` 表中 `id=1` 的管理员密码被改写为固定值，说明系统已经发生了明确的后台账户篡改事件。按照题目描述，这正是需要溯源的异常结果。

## 四、Web 侧关键证据

围绕数据库异常时间点前后筛查访问日志后，发现来源 `157.7.196.221` 在极短时间内多次向一个高度可疑的脚本路径发起 POST 请求。

| 时间 | 来源 IP | 请求路径 | 方法 | 判定 |
| --- | --- | --- | --- | --- |
| `27/Sep/2023:16:36:42 +0800` | `157.7.196.221` | `/uploads/upload/xiaoma.php` | `POST` | 可疑 Webshell 交互 |
| `27/Sep/2023:16:36:47 +0800` | `157.7.196.221` | `/uploads/upload/xiaoma.php` | `POST` | 可疑 Webshell 交互 |
| `27/Sep/2023:16:36:49 +0800` | `157.7.196.221` | `/uploads/upload/xiaoma.php` | `POST` | 可疑 Webshell 交互 |
| `27/Sep/2023:16:37:06 +0800` | `157.7.196.221` | `/uploads/upload/xiaoma.php` | `POST` | 与数据库篡改时间高度贴合 |

之所以这一组请求比其他扫描流量更有价值，原因有两点。其一，请求目标是 `xiaoma.php` 这类明显不符合正常业务命名习惯的脚本路径，典型指向后门或 Webshell。其二，这几次请求全部集中发生在数据库异常更新语句之前不到半分钟的时间窗口内，因此不仅“可疑”，而且能够解释异常结果为什么会出现。

## 五、时间线关联分析

将 Web 请求与数据库日志合并到同一时间轴后，可以得到如下攻击链。

| 时间 | 事件 | 含义 |
| --- | --- | --- |
| `16:36:42` | `157.7.196.221` 首次 POST `/uploads/upload/xiaoma.php` | 攻击者开始通过后门或恶意脚本执行操作 |
| `16:36:47` 至 `16:37:06` | 同一来源继续多次 POST | 持续执行指令或调整攻击载荷 |
| `16:37:10` | 数据库出现 `update admin set password="154444" where id=1` | 管理员密码被篡改 |

这个时间关系非常关键。数据库中的异常不是孤立发生的，而是紧跟在 `157.7.196.221` 与可疑 PHP 脚本的交互之后，因此可以将该来源地址认定为导致管理员账户异常的直接攻击者。

## 六、最终答案

综合数据库篡改结果、Webshell 请求特征与时间窗口关联性，本题最终提交为：

```text
flag{157.7.196.221}
```
