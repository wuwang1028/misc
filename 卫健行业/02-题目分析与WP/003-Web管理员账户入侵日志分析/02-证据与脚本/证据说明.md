# 003-Web管理员账户入侵日志分析证据说明

本题需要把数据库层面的异常结果与 Web 侧的攻击入口对应起来。当前最关键的证据是：数据库日志在 `2023-09-27 16:37:10` 记录到管理员密码被改写，而 Web 日志中来源 `157.7.196.221` 在这一时间窗口前连续多次向 `/uploads/upload/xiaoma.php` 发起 POST 请求，二者构成了清晰的因果关联。

## 关键数据库证据

```text
230927 16:37:10 ... update admin set password="154444" where id=1
```

## 关键 Web 证据

```text
27/Sep/2023:16:36:42 +0800 157.7.196.221 POST /uploads/upload/xiaoma.php
27/Sep/2023:16:36:47 +0800 157.7.196.221 POST /uploads/upload/xiaoma.php
27/Sep/2023:16:36:49 +0800 157.7.196.221 POST /uploads/upload/xiaoma.php
27/Sep/2023:16:37:06 +0800 157.7.196.221 POST /uploads/upload/xiaoma.php
```

## 结论摘要

| 要素 | 结论 |
| --- | --- |
| 导致管理员账户异常的攻击者 IP | `157.7.196.221` |
| 关键异常行为 | 对疑似 Webshell `xiaoma.php` 连续发起 POST 请求 |
| 数据库结果 | `admin` 表管理员密码被更新为 `154444` |
| 最终答案 | `flag{157.7.196.221}` |

## 说明

普通静态资源访问与常规页面浏览无法解释数据库中出现的密码篡改语句。只有 `157.7.196.221` 在异常时段与可疑 PHP 脚本的交互，能够同时满足“具备攻击入口”和“时间上紧邻异常结果”这两个条件。
