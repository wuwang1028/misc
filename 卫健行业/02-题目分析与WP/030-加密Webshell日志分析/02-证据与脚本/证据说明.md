# 030-加密Webshell日志分析：证据说明

## 一、核心结论

通过 `access.log` 中 `/font.php` 的多条成功攻击请求，可以确定攻击者实际使用的载荷密码为：

```text
5wxBnp5iX879uyP1d2js3MIOVrYFS76U
```

因此最终提交值为：

```text
flag{5wxBnp5iX879uyP1d2js3MIOVrYFS76U}
```

## 二、关键证据

| 证据项 | 内容 |
| --- | --- |
| 真实攻击入口 | `/font.php` |
| 访问方式 | 多条 `POST /font.php`，状态码均为 `200` |
| 固定参数名 | `5wxBnp5iX879uyP1d2js3MIOVrYFS76U` |
| 变动参数 | `m9ad948a8dfa85`、`te132822326775`、`x75801a1a4fa8d`、`nc6b66925b42c7` 等 |
| 判定依据 | 固定参数在多条不同功能载荷请求中持续复用，符合单行 Webshell 密码特征 |

## 三、代表性请求样本

以下请求体均以相同参数名开头：

```text
5wxBnp5iX879uyP1d2js3MIOVrYFS76U=...
```

其后载荷内容分别实现：

1. 列目录与文件信息。
2. 执行 `/bin/sh -c` 命令。
3. 回显环境与当前路径。

这说明该固定字符串并非普通业务字段，而是攻击者驱动所有后续行为的核心入口密码。

## 四、排除项说明

`/lang.php` 也有大量返回 200 的 POST 请求，但其请求体整体为密文样式，仅能证明存在加密会话，无法仅凭访问日志直接恢复口令。相比之下，`/font.php` 的固定参数名具备**直接可见、持续复用、语义明确**三项特征，因此应认定为题面要求的真实攻击载荷密码。
