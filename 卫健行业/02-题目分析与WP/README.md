# 题目分析与 WP 目录说明

本目录用于按题目统一归档分析材料。每一道题在本目录下应拥有独立子目录，并统一拆分为 **`01-分析与WP/`** 与 **`02-证据与脚本/`** 两层结构：前者保存正式 WriteUp，后者保存关键证据、验证输出、辅助脚本、源码恢复结果以及补充研究记录。

| 题目目录 | 内容说明 |
|---|---|
| `027-安全漏洞事件分析/` | 已按统一结构整理为正式 WP 与源码恢复、利用证据、辅助脚本分离归档 |
| `慢SQL优化/` | 慢查询定位、优化过程、修复验证结果与相关脚本 |
| `主机基线加固/` | 基线整改分析、配置修复、验证过程与证据文件 |
| `系统后门排查/` | 系统后门排查过程、取证结果、修复步骤与验证脚本 |
| `服务后门排查/` | 服务侧后门排查过程、计划任务修复、判题验证与证据归档 |

后续新增题目时，应继续沿用这一模式，将**单题分析材料**统一放入本目录；与之对应的**环境复现材料**则统一放入 `../03-复现环境/` 下的同名题目目录，以保持全仓库结构一致。
