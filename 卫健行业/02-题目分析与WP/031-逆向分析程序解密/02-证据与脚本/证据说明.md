# 031-逆向分析程序解密：证据说明

## 一、核心结论

通过对 `a.exe` 的 PyInstaller 拆包与 `CTF.pyc` 字节码分析，可以确定程序校验的最终 flag 为：

```text
flag{who_find_me_ohh}
```

## 二、关键证据

| 证据项 | 内容 |
| --- | --- |
| 样本类型 | PyInstaller 打包的 Python 程序 |
| 主逻辑文件 | `CTF.pyc` |
| 算法特征 | `func1` 为 RC4 KSA，`func2` 为 RC4 PRGA |
| 固定密钥 | `flag` |
| 密文数组 | `[152, 152, 187, 86, 173, 143, 37, 84, 229, 52, 176, 7, 31, 231, 128]` |
| 明文结果 | `who_find_me_ohh` |

## 三、程序校验逻辑摘要

程序先检查输入是否满足以下格式：

```text
flag{???????????????}
```

其中前 5 个字符必须为 `flag{`，第 21 个字符必须为 `}`，中间 15 个字符才是真正参与运算的部分。随后程序会：

1. 把中间 15 个字符转换为字节数组；
2. 使用硬编码密钥 `flag` 初始化 RC4 状态；
3. 将该 15 字节数据与 RC4 密钥流异或；
4. 与内置密文数组比较。

因此只要对内置数组做同样的 RC4 逆运算，即可直接恢复中间明文。

## 四、恢复结果

恢复出的中间 15 字节内容为：

```text
who_find_me_ohh
```

所以完整提交值为：

```text
flag{who_find_me_ohh}
```
