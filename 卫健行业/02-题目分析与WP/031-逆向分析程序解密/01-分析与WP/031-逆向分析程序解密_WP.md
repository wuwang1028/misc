# 031-逆向分析程序解密 WP

## 一、题目信息

| 项目 | 内容 |
| --- | --- |
| 题号 | 031 |
| 题目名称 | 逆向分析程序解密 |
| 题型 | 附件分析题 |
| 提交格式 | `flag{xxx}` |
| 题目目标 | 对给定可执行程序进行逆向分析，恢复程序校验的正确 flag |

## 二、样本识别

附件只有一个文件 `a.exe`。使用基础文件识别后可知其为 **Windows 64 位控制台程序**。继续做字符串检查，可以看到明显的 **PyInstaller** 特征，例如 `PyInst`、`PYI_ONEDIR` 等信息，因此样本并非原生 C/C++ 程序，而是一个被打包的 Python 程序。

这一步非常关键，因为一旦确认是 PyInstaller 样本，就不应直接在汇编层硬抠逻辑，而是优先拆包出其中的 Python 字节码，再进行高层逻辑分析。

## 三、拆包与字节码分析

### 1. 拆包得到主逻辑文件

使用 PyInstaller 提取脚本对 `a.exe` 做拆包后，得到了核心文件：

```text
CTF.pyc
```

同时还可以看到 `PYZ-00.pyz`、运行时 DLL、`base_library.zip` 等标准打包产物，由此进一步确认样本确实是 PyInstaller 打包的 Python 程序。

### 2. 反汇编 `CTF.pyc`

该样本由 **Python 3.10** 打包，因此使用 Python 3.10 对 `.pyc` 做字节码级反汇编。反汇编结果显示模块中共有三个关键函数：

| 函数名 | 作用 |
| --- | --- |
| `func1` | 初始化 256 字节状态表并进行密钥调度 |
| `func2` | 对输入数据逐字节异或处理 |
| `func` | 接收用户输入、取出中间内容、调用前两者并判断是否与内置密文一致 |

从 `func1` 和 `func2` 的结构可以明确识别出，这是一个标准的 **RC4** 实现：前者对应 **KSA**，后者对应 **PRGA**。

## 四、核心逻辑恢复

在 `func` 中可以看到程序的校验逻辑：

1. 程序要求用户输入一个形如 `flag{xxxxxxxxxxxxxxx}` 的字符串。
2. 程序逐字节检查前缀必须是 `f`、`l`、`a`、`g`、`{`，并检查第 21 个字节必须是 `}`。
3. 真正参与校验的是中间 15 个字符，即 `input_bytes[5:20]`。
4. RC4 的密钥被硬编码为字符串：

```text
flag
```

5. 目标密文数组为：

```text
[152, 152, 187, 86, 173, 143, 37, 84, 229, 52, 176, 7, 31, 231, 128]
```

程序会把用户输入中间的 15 个字符转为 `bytearray`，然后经过 RC4 运算，与上述数组比较。只要恢复出这 15 字节明文，就能得到最终 flag。

## 五、还原正确明文

按照样本中的相同 RC4 算法，使用密钥 `flag` 对内置密文数组进行逆向恢复，可以得到明文：

```text
who_find_me_ohh
```

因此完整答案为：

```text
flag{who_find_me_ohh}
```

## 六、结论

031 题的本质是一个很典型的 **PyInstaller + Python 字节码逆向 + RC4 还原** 题。关键不在于运行程序暴力尝试，而在于识别打包框架、提取 `CTF.pyc`、还原其校验逻辑，并用相同算法逆向求出中间明文。

最终答案：

```text
flag{who_find_me_ohh}
```
