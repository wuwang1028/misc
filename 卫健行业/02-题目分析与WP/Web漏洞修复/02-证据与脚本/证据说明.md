# Web漏洞修复证据说明

## 一、证据用途说明

本目录用于保存《Web漏洞修复》题目的关键现场记录、远端自动化脚本、最终采用的最小兼容修复版源码与构建产物，以及最终放旗的关键回显。由于本题真正的难点不只是识别漏洞类型，还包括在受限在线终端环境中把修复后的 JAR 稳定回传并重新部署，因此保留这条完整证据链，对于后续复盘、WP 撰写和环境复现设计都非常重要。

## 二、核心结论

本题的核心结论可以概括为三个层面。第一，题目要求的修复对象是 `java-upload-fix-1.0-SNAPSHOT.jar`，并且最终服务运行于 `/opt/app/` 下，由 `/restart-web.sh` 控制替换与重启。第二，业务形态是一个面向 `/doUpload` 的文件上传服务，题面提示修复重点为“文件控制不足”，因此漏洞本质可收敛为上传场景中的路径控制缺陷。第三，在原始 JAR 无法稳定导出的条件下，最终通过**最小兼容重建 + Base64 分片回传 + 受控重启**的方式完成修复，随后裁判成功生成 `/flag`。后续在动态环境复刷时，又确认新实例运行的是 OpenJDK 8，因此仓库内默认归档的修复包已更新为 **Java 8 兼容版本**，以避免因字节码版本过高导致服务启动失败。

| 结论项 | 结果 |
| --- | --- |
| 真实修复对象 | `java-upload-fix-1.0-SNAPSHOT.jar` |
| 关键业务接口 | `GET /`、`POST /doUpload` |
| 漏洞方向 | 文件控制不足导致的路径控制缺陷 |
| 最终修复方式 | 最小兼容替换 JAR |
| 最终结果 | `/flag` 成功生成 |
